# C++ Reorganization + `wire`→`message` Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Reorganize `cppbind/` into `cpp/{emitter,runtime,tests}`, base the runtime codec on `origin/master` (pack-side pad removed), rename `wire`→`message`, replace C error macros with a `zproto::errc` enum, and migrate the tests to the new API so the C++ tree builds and the C-reference byte-compat oracle runs.

**Architecture:** `emitter/` is the build-time code generator (links the C parser `../../zproto.c`). `runtime/` is a 2-file source-copy library (`zproto.hpp` + `zproto.cpp`, no Makefile, no `zproto.h` dependency). `tests/` builds the generator, generates `hello_world.{hpp,message.hpp}`, then builds + runs the functional round-trip (`test`) and the C oracle (`test_codec`). The codec in `runtime/zproto.cpp` is a near-verbatim copy of `origin/master`'s `zproto.c` pack/unpack, renamed `pack_impl`/`unpack_impl`, returning `zproto::errc` values.

**Tech Stack:** C++11 (`g++ -std=c++11 -Wall -Wextra`), C (`gcc`) for the reference `zproto.c`, GNU `make`.

## Global Constraints

- Compiler flags: `g++ -std=c++11 -Wall -Wextra -O2 -g -I../` for the runtime/tests; `gcc -Wall -g -O2 -c` for `zproto.c`. C++11 is required (`enum class` with underlying type, `constexpr`).
- **Runtime is C-free:** `cpp/runtime/zproto.hpp` and `cpp/runtime/zproto.cpp` MUST NOT `#include "zproto.h"`. Error codes live in `namespace zproto { enum class errc : int { oom = -1, error = -3 }; }` (no `NOFIELD` — unused in the runtime). `pack`/`unpack` return `int`; on failure they return `static_cast<int>(errc::…)`.
- **Codec base = `origin/master` (`cbecf36`).** The runtime codec copies `origin/master`'s `zproto.c` pack/unpack (pack-side pad removed: `packff` emits `sn` bytes, bound uses `srcsz`, no `sz8`). The current `perf-pack` working-tree `zproto.c` is the divergent pad-present version and must be aligned to `origin/master` first (Task 1).
- **`wire`→`message` rename** (pure): trait `wire<T>`→`message<T>`, generated file `X.wire.hpp`→`X.message.hpp`, all generator (`body.cpp`) emissions updated. Method signatures and behavior unchanged. Namespace stays `zproto`.
- **All code comments in English.**
- **Source-copy distribution:** `runtime/` ships only `zproto.hpp` + `zproto.cpp`; consumers copy both into their project. No `libzproto.a`, no `make install`, no Makefile in `runtime/`.
- **`bench/cpp` is NOT touched** (redone later with the Rust/Lua cross-comparison).
- **Commits are throwaway on `perf-pack`** (the user will regenerate one clean commit on `master` later), but each task still ends with a checkpoint commit for rollback during execution.

#include <assert.h>

namespace zproto {

// ---- zero-suppression codec: near-verbatim copy of zproto.c (origin/master,
// cbecf36): packseg/packff/pack/unpackseg/unpack. Static + renamed drivers
// (pack_impl/unpack_impl) to avoid clashing with zproto::pack/zproto::unpack.
// Error codes mapped to zproto::errc. Re-sync from zproto.c by copy, then
// re-apply: (1) make all fns static, (2) rename pack->pack_impl, unpack->
// unpack_impl, (3) ZPROTO_OOM -> (int)errc::oom, ZPROTO_ERROR -> (int)errc::error.
// Do NOT copy zproto_pack/zproto_unpack (thin bound-check shells; the Buffer
// wrappers below own sizing). pack does NOT pad the wire bytes (master base);
// the bound uses srcsz directly (no sz8 round-up).

// <CODEC BLOCK INSERTED IN STEP 5>

int pack(const uint8_t *src, int srcsz, Buffer &dst) {
    // master's bound: pack emits <= srcsz + overhead (no pad), so no sz8 round-up.
    assert(dst.data() == nullptr || src != (const uint8_t *)dst.data());
    int needn = ((srcsz + 2047) / 2048) * 2 + srcsz + 1;
    dst.resize((size_t)needn);
    int n = pack_impl(src, srcsz, (uint8_t *)dst.data());
    dst.set_size((size_t)n);
    return n;
}

int unpack(const uint8_t *src, int srcsz, Buffer &dst) {
    // bound: the caller's max (set_max — bomb defense) if set, else the loose 8x.
    size_t bound = (dst.max != SIZE_MAX) ? dst.max : ((size_t)srcsz * 8 + 8);
    if (!dst.ensure(bound))
        return (int)errc::oom;
    int n = unpack_impl(src, srcsz, (uint8_t *)dst.data(), (int)bound);
    if (n < 0) {
        dst.clear();
        return n;
    }
    dst.set_size((size_t)n);
    return n;
}

}  // namespace zproto
```

test_normal()
{

    struct zproto_parser p;
    assert(zproto_parse(&p, "hello.world.zproto" /* read file first */) == 0);
    // ... read the schema file into a buffer, zproto_parse it, query "packet" ...
    struct zproto *z = p.z;
    struct zproto_struct *packet = zproto_query(z, "packet");
    assert(packet);

    hello::world::packet pa;
    pa.address = "ShangHai"; pa.ii = 6; pa.ff = 7.0f; pa.ll = 8; pa.bb = true;
    pa.luck = {3, 7, 5};
    pa.address1 = {"hello", "world"};

    // oracle
    uint8_t ref[4096];
    int refsz = zproto_encode(packet, ref, sizeof(ref), enc_cb_pa, &pa);
    assert(refsz > 0);

    // new
    Buffer out;
    message<hello::world::packet>::encode(pa, out);
    assert((int)out.size() == refsz);
    assert(memcmp(out.data(), ref, refsz) == 0);

    zproto_free(z);
    printf("test_encoder ok\n");
}
```
`enc_cb_pa` is a C callback that serves `pa`'s fields by tag (mirror of the old `enc_cb`, but reading from a `hello::world::packet*`). The tag numbers come from the schema (1=`phone` map, 3=`address`, 4=`luck`, 5=`address1`, 6=`ii`, 7=`ff`, 8=`ll`, 9=`bb`, …) — read them from the parsed `zproto_struct` rather than hardcoding, to stay correct if the schema changes.

`test_decoder` mirrors this: feed `ref` (the C-encoded bytes) to `message<hello::world::packet>::decode` and assert each decoded field equals `pa`.

> Note: matching the C oracle field-by-field requires the callback to emit fields in the same order/tags the generated `message<T>::write_at` does. The generated deltas come from the schema's tag order, so driving `zproto_encode` with the same tags yields byte-identical output. If exact byte-match proves fragile for the map-typed `phone` field (unordered_map iteration order is non-deterministic), restrict `test_encoder`/`test_decoder` to the deterministic scalar/vector/string fields and keep the full map round-trip in `test.cpp` (Task 4). The byte-compat guarantee for pack/unpack (Step 2) is unaffected.
